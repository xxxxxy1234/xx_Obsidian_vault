


# 工业角焊缝视觉定位系统

**Python + OpenCV 传统视觉，无深度学习**  
数据来源：RGB图（2400×1800）+ PLY有序点云（含相机内参）

---

## 核心结论（对话精华）

### 数据真相

|数据|实际情况|可用性|
|---|---|---|
|PLY xyz坐标|75% 有效，25% NaN（金属反光区测量失败）|✅ 核心数据|
|PLY rgb字段|就是RGB图本身（像素一一对应）|辅助|
|PLY camera element|完整相机内参（focal/cx/cy/k1/k2）|可用于畸变校正|
|TIFF深度图|基本全255，无效|❌ 弃用|

### PLY格式本质

```
PLY = 结构化表格，4320000行 × 7列（x y z red green blue alpha）
读入Python → 7个numpy数组，shape=(4320000,)
z.reshape(1800, 2400) → 深度图，与RGB像素一一对齐

Z双峰分布：
  平面A（近板） ≈ 324mm  →  主峰1
  平面B（远板） ≈ 342mm  →  主峰2
  焊缝交线     ≈ 333mm  →  两峰之间的谷（深度不连续带）
  两板深度差：17.6mm
```

### 边框误检问题根因与解法

**根因**：板材边框在groove_mask和深度梯度图里都是亮线，与焊缝无法通过强度区分。

**正确解法**：跨平面边缘过滤。

- 焊缝两侧Z值不同（324 vs 342mm）→ 保留
- 边框两侧Z值相同（同一平面内的几何边）→ 丢弃

这是天然区分条件，不依赖任何角度假设，换什么角度的焊缝都能用。

### 各版本核心教训

|版本|失败原因|
|---|---|
|v4–v6|fitLine/HoughLinesP被板材边框像素主导，全局拟合被拉偏|
|v5 梯度方向掩码|金属反光导致Sobel方向杂乱，有效像素被误杀（suppress_mask全白噪声）|
|v6 HoughLinesP|ROI=250px时minLineLength=75px门槛过高；used_tol强制收到15°误杀线段|
|v7–v8 黑帽|方向正确，但端点延伸到ROI边界而非焊缝实际端点；板材边框暗侧也被黑帽提取|

## 主方案：`weld_seam_from_ply.py`

### 算法流程

```
PLY xyz → reshape(1800,2400) → Z深度图
         ↓
双峰检测 → zA≈324mm，zB≈342mm（自动，无需手动指定）
         ↓
Sobel梯度 → 高梯度边缘像素（top N%）
         ↓
跨平面判断（每个边缘像素沿法向采样两侧Z值）：
    左侧Z≈zA 且 右侧Z≈zB → 保留（焊缝边缘）
    两侧Z差 < z_margin   → 丢弃（板材边框）
         ↓
形态学清理（去孤立噪点）
         ↓
RANSAC fitLine（鲁棒拟合，排除残留噪点）
         ↓
骨架端点定位（沿主方向扫描mask，找实际起止点）
         ↓
输出：端点坐标、角度、交点、result.png、result.json
```

### 快速开始

```bash
pip install plyfile opencv-python numpy

# 不加ROI先看全局效果
python weld_seam_from_ply.py \
  --ply point_cloud_00000.ply \
  --rgb rgb_image_right_00000.png \
  --output result.png \
  --debug-dir ./debug/ \
  --json-out result.json \
  --no-display

# 加ROI精确检测
python weld_seam_from_ply.py \
  --ply point_cloud_00000.ply \
  --rgb rgb_image_right_00000.png \
  --roi 700,700,250,250 \
  --output result.png \
  --debug-dir ./debug/ \
  --json-out result.json \
  --no-display
```

### 参数说明

|参数|默认值|说明|
|---|---|---|
|`--ply`|必填|PLY点云路径|
|`--rgb`|None|RGB图路径（叠加结果用，可选）|
|`--roi`|全图|x,y,w,h，不填则处理全图|
|`--z-margin`|2.0|跨平面判断的Z容差(mm)，建议1.5~4.0|
|`--grad-pct`|85|梯度阈值百分位，越高越严格|
|`--ransac-iter`|500|RANSAC迭代次数|
|`--ransac-dist`|3.0|RANSAC内点距离阈值(px)|
|`--max-seams`|2|最多检测焊缝条数|

### 调试图说明

|文件|看什么|期望结果|
|---|---|---|
|`depth_map.png`|Z深度伪彩色|两块金属板颜色明显不同|
|`cross_plane_mask.png`|跨平面过滤原始结果|只有焊缝是白色，边框是黑色|
|`cross_plane_clean.png`|形态学清理后|干净的焊缝线条|
|`seam_detected.png`|绿线+端点叠加|绿线贴着焊缝，端点在焊缝两端|

### 调参指南

**焊缝检测不到：**

- 降低 `--grad-pct`（85→80），捕获更多边缘像素
- 扩大 `--z-margin`（2.0→3.0），跨平面判断更宽松

**边框仍被误检：**

- 缩小 `--z-margin`（2.0→1.5），要求两侧Z差更显著
- 先看 `cross_plane_mask.png`，确认白色区域是否只含焊缝

**端点不准：**

- `cross_plane_clean.png` 里mask在焊缝两端是否有断裂
- 调整 `--z-margin` 使mask更连续，或增大 `--ransac-dist`

## 备用方案：`weld_seam_pro_v9.py`（纯RGB）

PLY不可用时使用。核心逻辑：黑帽多核投票提取暗槽 + 轮廓几何过滤排除边框。

```bash
python weld_seam_pro_v9.py \
  --input rgb_image_right_00000.png \
  --roi 700,700,250,250 \
  --bh-kernels 7,11,15 \
  --bh-thresh 12 \
  --min-aspect 3.0 \
  --border-margin 8 \
  --output result.png \
  --debug-dir ./debug/ \
  --json-out result.json \
  --no-display
```

**边框排除逻辑（无角度假设）：**

1. 轮廓 `minAreaRect` 长宽比 < `min-aspect` → 团块噪点，丢弃
2. 轮廓bbox跨越ROI边界比例 > 60% 且紧贴边界 → 判定为板材边框，丢弃
3. 通过过滤的轮廓像素 → RANSAC fitLine → 骨架端点定位

调试图先看 `07_groove_mask_filtered.png`，白色区域应只含焊缝。

## 重要约束

1. **不硬编码角度**：焊缝角度未知，两套方案均不依赖角度先验，换场景直接用
2. **ROI坐标通用**：PLY有序点云与RGB图像素一一对应，ROI坐标直接复用
3. **numpy 2.x兼容**：sklearn/scipy与numpy 2.x不兼容，代码全部用纯numpy替代
4. **端点定位**：骨架扫描法（沿fitLine方向找mask最远白色像素），不延伸到ROI边界
5. **TIFF弃用**：金属反光导致深度传感器失效，TIFF全255无参考价值，统一用PLY
